# The Diplomat v1.1 — Design Document

**Date:** 2026-04-03
**Status:** Approved
**Feature:** Per-vassal delegation of process control from King daemon to Claude MCP session

---

## Overview

The Diplomat enables a two-phase handoff model where a Claude Code session can observe and optionally take full control of individual vassals managed by a King daemon. This allows Claude to restart, debug, and manage specific processes without conflicting with the daemon's self-healing logic.

---

## Two-Phase Delegation Model

### Phase 1: Silent Observation (automatic on `king mcp` startup)

When the MCP server starts in a directory that is a subdirectory of a registered kingdom:

1. MCP server reads `~/.king/registry.json` to detect a parent kingdom
2. Enters **Observer Mode** — can read logs, query events, but daemon retains full control
3. Informs Claude: `"I'm inside kingdom /projects/root. Vassal 'ui' is managed by King daemon. Mode: Observation."`

No delegation occurs automatically. The daemon continues all restarts and monitoring.

### Phase 2: Full Takeover (explicit via MCP tool)

When Claude needs to act on a vassal:

1. Claude calls `delegate_control(vassal="ui", force=false)`
2. MCP server sends RPC to parent kingdom daemon socket
3. Daemon marks vassal as `DELEGATED` in-memory
4. Daemon's `watchVassal()` goroutine skips restart decisions for delegated vassals
5. MCP server starts background heartbeat goroutine (10s interval)

## Architecture

### Delegation State — In-Memory in Daemon

Transient state, not persisted. If the daemon restarts, all delegation is released (MCP server detects this via heartbeat response and re-delegates automatically).

```go
type DelegationInfo struct {
    SessionPID    int
    LastHeartbeat time.Time
}

// In Daemon struct:
delegatedVassals map[string]DelegationInfo  // key: vassal name
delegationMu     sync.RWMutex
```

### New Daemon RPC Methods

| Method | Params | Returns | Description |
|--------|--------|---------|-------------|
| `delegate_control` | `{vassal string, force bool}` | `{ok bool, error string}` | Grant control to caller |
| `delegate_heartbeat` | `{vassal string, session_pid int}` | `{acknowledged bool, status string}` | Update liveness timestamp |
| `delegate_release` | `{vassal string}` | `{ok bool}` | Explicit return of control to daemon |

**`delegate_heartbeat` response semantics:**
- `acknowledged: true` — daemon knows about this delegation, updated timestamp
- `acknowledged: false` — daemon has no record (restarted) → MCP client should re-call `delegate_control`

### New MCP Tools (exposed to Claude)

| Tool | Params | Description |
|------|--------|-------------|
| `delegate_control` | `vassal string, force bool` | Request full control over a vassal |
| `delegate_release` | `vassal string` | Return control to daemon |
| `delegate_status` | — | Show current delegation state for all vassals |

### MCP Server Changes

**`detectParentKingdom()`** — called during server initialization:

```go
func (s *Server) detectParentKingdom() {
    reg := registry.NewRegistry(registryPath)
    entries, _ := reg.ListAlive()
    for path, entry := range entries {
        if strings.HasPrefix(s.rootDir, path) && s.rootDir != path {
            s.parentKingdomSocket = entry.Socket
            s.inObserverMode = true
            break
        }
    }
}
```

**Heartbeat goroutine** — starts after first successful `delegate_control`:

```go
go func() {
    ticker := time.NewTicker(10 * time.Second)
    for range ticker.C {
        resp := client.Call("delegate_heartbeat", {vassal, os.Getpid()})
        if !resp.Acknowledged {
            // Daemon restarted — re-delegate
            client.Call("delegate_control", {vassal, false})
        }
    }
}()
```

### Daemon Changes

**`watchVassal()` guard** — skip restart if delegated:

```go
func (d *Daemon) watchVassal(name string, ...) {
    for {
        cmd.Wait() // process exited
        if d.isDelegated(name) {
            d.logger.Info("VASSAL_DELEGATED_EXIT", "vassal", name)
            return // Claude has control, don't restart
        }
        // normal backoff restart logic...
    }
}
```

**`runDelegationWarden()`** — background goroutine, runs every 10s:

```go
func (d *Daemon) runDelegationWarden() {
    ticker := time.NewTicker(10 * time.Second)
    for range ticker.C {
        d.delegationMu.Lock()
        for vassal, info := range d.delegatedVassals {
            if time.Since(info.LastHeartbeat) > 30*time.Second {
                delete(d.delegatedVassals, vassal)
                d.logger.Warn("DELEGATION_EXPIRED",
                    "vassal", vassal,
                    "session_pid", info.SessionPID)
                d.maybeRestartOrphanedVassal(vassal)
            }
        }
        d.delegationMu.Unlock()
    }
}
```

**`maybeRestartOrphanedVassal()`** — "The Great Return":
- If vassal process is alive → daemon takes over monitoring only
- If vassal process is dead → daemon initiates backoff restart (5s → 15s → 60s)
- Emits event: `WARNING: Control resumed after session crash (PID <old>)`

## Data Flow

```
Claude Code → king mcp (stdio process)
    │
    ├─ startup: read registry → detect parent kingdom
    ├─ notify Claude: "Observer mode active for kingdom X"
    │
    │  [user asks Claude to fix 'ui']
    │
    ├─ delegate_control("ui") →
    │     RPC → parent daemon socket →
    │     daemon.delegatedVassals["ui"] = DelegationInfo{...}
    │     watchVassal("ui") now skips restarts
    │
    ├─ heartbeat goroutine (every 10s) →
    │     RPC → daemon.updateHeartbeat("ui")
    │     if !acknowledged → re-delegate (daemon restart recovery)
    │
    │  [Claude session exits / crashes]
    │
    └─ heartbeat stops →
          Warden goroutine detects staleness (>30s) →
          daemon reclaims "ui":
            - if process alive → monitor only
            - if process dead → backoff restart
          log: "DELEGATION_EXPIRED vassal=ui session_pid=1550"
```

## Edge Cases

| Scenario | Behavior |
|----------|----------|
| Two Claude sessions try to delegate same vassal | Second call returns error unless `force=true` |
| Daemon restarts while Claude is active | Heartbeat returns `acknowledged=false` → MCP auto re-delegates |
| Claude exits gracefully (not crash) | `delegate_release` called in MCP server shutdown hook |
| Vassal process crashes while delegated | Daemon warden does NOT restart (Claude has control); Claude detects via `list_vassals` |
| Delegated vassal + no parent kingdom detected | `delegate_control` returns error: "no parent kingdom found" |
